1. Как это работает (общая схема)

Сайт ФТО — статичный, работает на GitHub Pages без серверной базы данных. Поэтому обновление каталога устроено так:

  1. Вы открываете админку в браузере и вносите изменения
  2. Админка сохраняет данные локально (в браузере)
  3. Когда готовы — нажимаете кнопку «Скачать все товары»
  4. Получаете ZIP-архив с JSON-файлами
  5. Распаковываете архив и копируете файлы в репозиторий на GitHub
  6. Через 1-2 минуты каталог на сайте обновляется автоматически
Преимущества такого подхода: не нужен сервер, не нужна база данных, нет риска потерять данные при сбое. Все товары хранятся прямо в репозитории Git — это значит, что у вас есть история изменений и можно откатить любую правку.

2. Шаг 1: Открыть админку

Откройте в браузере адрес: https://alfat81.github.io/fto/admin.html

Или локально, если вы работаете с копией сайта: docs/admin.html

Админка работает в любом современном браузере (Chrome, Firefox, Edge, Safari). Рекомендуется Chrome.

Важно: данные хранятся в localStorage браузера. Если вы очистите кэш браузера или поменяете устройство — изменения пропадут. Поэтому регулярно скачивайте JSON-архив.

3. Шаг 2: Добавить новый товар

  1. Нажмите кнопку «Добавить товар» (вверху страницы или в блоке «Товары в каталоге»)
  2. Заполните форму:
    • ID товара * — уникальный идентификатор. Примеры: 5.1, 5.2, stallazh-150, kassa-usb. Используется в имени файла и для изображения.
    • Название * — что увидит покупатель. Например: «Стеллаж торговый СТ-150».
    • Категория * — выберите из списка:
      • 1 — Торговые стеллажи
      • 2 — Витрины и прилавки
      • 3 — Кассовые боксы
      • 4 — Кресла и стулья
    • Цена, ₽ * — числовое значение, например: 4500. Без пробелов и символов.
    • Старая цена — заполните, если хотите показать скидку. Например, цена 4500, старая цена 6000 — будет видно «−25%».
    • URL изображения — путь к картинке, например: images/5.1.jpg. Если оставить пустым, система будет искать images/{ID}.jpg автоматически.
    • Описание — краткое описание для карточки товара и SEO. 1-3 предложения.
    • Характеристики — пары «параметр — значение». Нажмите «Добавить характеристику» и заполните. Например:
      • «Высота» — «1800 мм»
      • «Ширина» — «900 мм»
      • «Нагрузка на полку» — «150 кг»
      • «Материал» — «Металл + ЛДСП»
      • «Цвет» — «Серый RAL 7035»
  3. Нажмите «Сохранить» — товар появится в списке справа.

4. Шаг 3: Редактировать товар

  1. Найдите нужный товар в списке справа
  2. Нажмите синюю кнопку (редактировать)
  3. Внесите изменения в форму слева
  4. Нажмите «Сохранить»

Рядом с изменённым товаром появится оранжевая метка «● изменён» — так вы видите, какие товары нужно будет скачать.

5. Шаг 4: Удалить товар

  1. Нажмите красную кнопку (удалить) рядом с товаром
  2. Подтвердите удаление в диалоговом окне
Внимание: удалённый товар нельзя восстановить в админке. Если удалили случайно — просто добавьте заново через «Добавить товар».

6. Шаг 5: Скачать JSON и обновить сайт

Когда все изменения внесены:

  1. Нажмите кнопку «Скачать все товары (ZIP-структура)» вверху страницы
  2. Браузер скачает файл с именем вида fto-catalog-2025-06-29.zip
  3. Распакуйте архив — внутри будет папка products/ с JSON-файлами и README.txt
  4. Теперь нужно загрузить файлы в репозиторий GitHub. Два варианта:
    • Вариант A (через сайт GitHub): откройте github.com/alfat81/fto → перейдите в docs/data/products/ → удалите все старые .json файлы → загрузите новые (drag & drop)
    • Вариант B (через Git): склонируйте репозиторий, замените файлы в docs/data/products/, сделайте commit и push (см. README.txt внутри архива)
  5. Через 1-2 минуты откройте catalog.html — каталог обновлён!
Подсказка: в архиве также есть обновлённый index.json — это список всех файлов. Обязательно загрузите и его тоже, иначе новые товары не появятся в каталоге.

7. Изображения товаров

Каждый товар может иметь изображение. Логика такая:

  1. Если в JSON-файле указано поле "image" — используется указанный путь. Например: "image": "images/my-photo.jpg"
  2. Если поле "image" не указано — система автоматически ищет images/{ID}.jpg. Например, для товара с ID 5.1 будет использоваться images/5.1.jpg

Как добавить изображение:

  1. Подготовьте фото в формате JPG, размер не более 200 КБ (оптимально — 800×1000 пикселей, портретная ориентация)
  2. Назовите файл по ID товара: 5.1.jpg, 5.2.jpg и т.д.
  3. Загрузите файл в папку docs/images/ репозитория на GitHub
Формат и качество: для каталога лучше всего подходят фотографии на белом фоне. Если фото нет — будет показана картинка-заглушка «нет фото».

8. Рекомендации по заполнению

Название товара

Должно быть информативным и содержать ключевые слова для SEO:

  • ❌ Плохо: «Кресло 1»
  • ✅ Хорошо: «Кресло офисное Вель СН-800 Ткань черная»
  • ✅ Хорошо: «Стеллаж торговый пристенный СТ-150 (1800×900×400)»

Описание

1-3 предложения с ключевыми характеристиками. Помогает SEO и покупателю принять решение:

  • ❌ Плохо: «Хороший стеллаж»
  • ✅ Хорошо: «Надежный сборно-разборный стеллаж для магазина и склада. Металлический каркас, нагрузка до 150 кг на полку. Порошковая покраска по RAL.»

Характеристики

Чем больше — тем лучше. Основные параметры, которые ищут покупатели:

  • Размеры: Высота, Ширина, Глубина (в мм)
  • Нагрузка: на полку, на секцию (в кг)
  • Материалы: металл, ЛДСП, стекло, массив
  • Цвет: по каталогу RAL или словами («серый», «черный»)
  • Комплектация: количество полок, наличие ящиков
  • Гарантия: месяцев
  • Срок изготовления: дней

Цены

  • Указывайте цену за единицу товара в рублях без копеек
  • Если цена зависит от размера/цвета — указывайте минимальную и помечайте «от» в описании
  • Для скидок — заполняйте «Старая цена», система покажет зачёркнутую цену автоматически

Категории

Если вам нужна новая категория (например, «Холодильное оборудование»), обратитесь к разработчику — нужно будет добавить её в products-loader.js и в навигацию каталога.

9. Частые проблемы

«Я добавил товар, но он не появился на сайте»

Проверьте, что вы:

  1. Скачали ZIP-архив кнопкой «Скачать все товары»
  2. Загрузили ВСЕ файлы из архива (включая index.json) в docs/data/products/ на GitHub
  3. Удалили старые JSON-файлы, которых больше нет в архиве
  4. Очистили кэш браузера (Ctrl+F5) на странице каталога

«Картинка товара не отображается»

Проверьте, что:

  1. Файл изображения загружен в docs/images/
  2. Имя файла совпадает с ID товара: 5.1.jpg для ID 5.1
  3. Расширение — именно .jpg (не .jpeg или .png)

«Случайно удалил все товары»

Не паникуйте — данные в репозитории GitHub не пострадали. Просто импортируйте JSON-файлы заново:

  1. Скачайте все JSON-файлы из docs/data/products/ на GitHub
  2. В админке нажмите «Импорт JSON» и выберите все файлы (можно несколько сразу)
  3. Товары загрузятся в админку

«Не работает кнопка скачать»

Проверьте, что браузер не блокирует скачивание. В Chrome: Настройки → Конфиденциальность → Файлы → разрешить автоматические загрузки для этого сайта.

«Изменения пропали после перезагрузки страницы»

Проверьте, что вы используете тот же браузер и тот же профиль. localStorage привязан к конкретному браузеру. Если открыли админку в режиме «Инкогнито» — данные не сохранятся.

10. Альтернативный способ: через GitHub

Если вы уверенно работаете с Git, можно редактировать JSON-файлы напрямую:

  1. Откройте папку docs/data/products/ на GitHub
  2. Нажмите «Add file → Create new file»
  3. Введите имя: 5.1.json
  4. Вставьте JSON-структуру (пример ниже)
  5. Нажмите «Commit new file»
  6. Отредактируйте index.json — добавьте имя нового файла в список

Пример структуры JSON-файла товара:

{
  "id": "5.1",
  "category": 1,
  "name": "Стеллаж торговый пристенный СТ-150",
  "price": 4500,
  "old_price": 6000,
  "description": "Надежный сборно-разборный стеллаж для магазина и склада. Металлический каркас, нагрузка до 150 кг на полку.",
  "specs": {
    "Высота": "1800 мм",
    "Ширина": "900 мм",
    "Глубина": "400 мм",
    "Нагрузка на полку": "150 кг",
    "Количество полок": "5 шт",
    "Материал": "Металл",
    "Цвет": "Серый RAL 7035"
  }
}

Минимальный набор полей:

{
  "id": "5.1",
  "category": 1,
  "name": "Стеллаж торговый СТ-150",
  "price": 4500
}
Совет: для массового добавления (10+ товаров) удобнее использовать админку — она автоматически формирует index.json и валидирует структуру.
